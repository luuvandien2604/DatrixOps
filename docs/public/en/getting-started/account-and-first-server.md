The goal of this guide is to create a server record and obtain the installation command that belongs to that server.

## Prerequisites

- Access to the DatrixOps application.
- Administrative access to the machine where the Agent will run.
- Outbound HTTPS access to `datrixops.vandien.space`.

## Create an account and sign in

1. Open **Register**, enter an email and password, and complete registration.
2. Open **Login** and sign in.
3. When the Dashboard loads, open **Servers**.

The web session uses an access token and a refresh token. When the access token expires, the API client attempts a refresh before asking you to sign in again.

## Add a server

1. Select **+ Add Server**.
2. Enter a recognizable name such as `api-production-01`.
3. Select Linux, macOS, or Windows.
4. Confirm the new server.
5. Copy the exact installation command generated by the Dashboard.

The command contains a unique Agent Token. Do not paste it into a public chat, support ticket, screenshot, or Git repository.

> **Warning:** Documentation examples use `<AGENT_TOKEN>`. Always use the token generated by the Dashboard for the intended server.

## Expected result

The server may remain **Offline** before installation. After a successful installation, the first heartbeat normally changes it to **Online** shortly afterward. IP address, operating system, and resource data then begin to appear.

If the server does not come online, continue with [Install the Agent](/docs/en/getting-started/installation) and [Troubleshooting](/docs/en/troubleshooting/common-issues).
