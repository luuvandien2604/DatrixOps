# Agent Release Process

## Release contract

Use one version value for both sides:

- `scripts/publish-agent.sh` embeds `AGENT_VERSION` into all release binaries.
- The backend reads `AGENT_VERSION` and reports whether an agent is outdated.
- A version mismatch is informational. Updates only run after an approved
  `agent_update` task from **Update Agent** or **Update all agents**.

## Publish

```bash
export AGENT_VERSION=1.3.0
bash scripts/publish-agent.sh
```

The script builds Linux amd64/arm64, macOS amd64/arm64, and Windows amd64
binaries into `frontend/public`. Installer scripts are source-controlled and
are not regenerated by the publish script.

## Deploy

Set the same backend version:

```bash
export AGENT_VERSION=1.3.0
docker compose -f docker-compose.prod.yml build backend frontend
docker compose -f docker-compose.prod.yml up -d backend frontend
```

Deploy the backend before queueing updates so task migrations and the bulk
endpoint are available. Deploy the frontend so every binary URL serves the new
release.

## Roll out

1. Update one online canary agent.
2. Confirm it reconnects and reports the new version.
3. Select **Update all agents**.
4. Review the queued/skipped result and agent logs.

Update tasks remain claimable for 24 hours. Offline agents can update after
reconnecting. Agents with an update already pending or processing are skipped.

## Update behavior

The current agent downloads the binary for its exact OS and architecture,
rejects empty or invalid executable formats, and stages it beside the running
binary. Linux and macOS replace the executable atomically before the service
manager restarts it. Windows uses a short-lived PowerShell helper to replace the
locked executable after the old process exits, then starts the Scheduled Task.

The installer path remains compatible with legacy agents for the first upgrade.
It downloads to a temporary file before replacing the binary and does not stop
the updater before the download completes.

## Checklist

- [ ] `AGENT_VERSION` is the same for publish and backend.
- [ ] All five binaries exist in `frontend/public`, including Windows.
- [ ] Backend and frontend images were rebuilt after publishing.
- [ ] Database migrations completed on backend startup.
- [ ] One canary agent successfully reconnected on the new version.
- [ ] Bulk rollout result matches the expected fleet size.
